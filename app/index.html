sg32 connects to Solana's gossip network, discovers validators, and requests block data via the repair protocol. It recovers the full block using Reed-Solomon erasure coding, builds a Merkle tree over the transactions, and serves verifiable inclusion proofs over HTTP.

API

MethodParamsReturns
getLatestSlot(none)slot, merkle_root, num_transactions
getSlot[slot]slot, parent_slot, merkle_root, num_transactions

getProof[slot, tx_index]leaf, proof[], root, verified

Quick Start

Run

./sg32

Check latest slot

curl -X POST http://localhost:8899/jsonrpc \
  -H 'Content-Type: application/json' \
  -d '{"jsonrpc":"2.0","method":"getLatestSlot","id":1}'

Get a Merkle proof

curl -X POST http://localhost:8899/jsonrpc \
  -H 'Content-Type: application/json' \
  -d '{"jsonrpc":"2.0","method":"getProof","params":[466680759, 0],"id":1}'

The response includes a verified field. If true, the transaction at that index is cryptographically proven to be in the block.

Python python

import requests, json

RPC = "http://localhost:8899/jsonrpc"
HEADERS = {"Content-Type": "application/json"}

def get_proof(slot: int, tx_index: int) -> dict:
    resp = requests.post(RPC, json={
        "jsonrpc": "2.0",
        "method": "getProof",
        "params": [slot, tx_index],
        "id": 1
    }, headers=HEADERS)
    return resp.json()["result"]

proof = get_proof(466680759, 0)
if proof["verified"]:
    print(f"Transaction is in slot {proof['slot']}")
    print(f"Merkle root: {proof['root']}")
else:
    print("Proof verification failed")

JavaScript js

const RPC = "http://localhost:8899/jsonrpc";

async function getProof(slot, txIndex) {
  const res = await fetch(RPC, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({
      jsonrpc: "2.0",
      method: "getProof",
      params: [slot, txIndex],
      id: 1
    })
  });
  const { result } = await res.json();
  return result;
}

getProof(466680759, 0).then(proof => {
  if (proof.verified) {
    console.log(`Transaction is in slot ${proof.slot}`);
  }
});

Rust rust

use serde_json::json;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let client = reqwest::Client::new();
    let resp = client
        .post("http://localhost:8899/jsonrpc")
        .json(&json!({
            "jsonrpc": "2.0",
            "method": "getProof",
            "params": [466680759, 0],
            "id": 1
        }))
        .send()
        .await?
        .json::<serde_json::Value>()
        .await?;
    let verified = resp["result"]["verified"].as_bool().unwrap_or(false);
    println!("verified: {}", verified);
    Ok(())
}

For Relayers and Bridges

Your relayer or bridge can verify a transaction's inclusion in a Solana block without trusting any third party:

flow:
  1. User submits transaction to Solana
  2. Your relayer detects the tx in a confirmed block
  3. Your relayer calls getProof(slot, tx_index) on sg32
  4. It checks proof.verified == true
  5. It independently verifies the Merkle root matches
     the block hash from a trusted source (e.g. your own
     light client, another sg32 node, or a public explorer)
  6. If both match, the transaction is provably included

No RPC provider to trust. No full node to run. One binary, one port.
